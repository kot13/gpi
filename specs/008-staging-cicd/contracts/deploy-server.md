| Component | Version | Notes |

## Directory layout

```text

├── .env                         # NOT in git — server only
├── .next/                       # generated by build
├── logs/
│   └── deploy.log               # append deploy history
├── node_modules/
├── public/
│   └── media/                   # persisted uploads (gitignored)
├── scripts/deploy/

├── ecosystem.config.cjs         # PM2 config (in repo)

```

## PM2 ecosystem (`ecosystem.config.cjs`)

```javascript
module.exports = {
  apps: [
    {
      name: 'gpi-staging',
      cwd: '/var/www/gpi',
      script: 'npm',
      args: 'run start',
      env: {
        NODE_ENV: 'production',
        PORT: 3000,
      },
      instances: 1,
      autorestart: true,
      max_memory_restart: '1G',
    },
  ],
}
```

**First-time setup** (manual):

```bash
cd /var/www/gpi
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup   # systemd integration for reboot persistence
```

## `scripts/deploy/staging-remote.sh`

Executable bash script; runs **on VPS** (invoked via SSH from GitHub Actions).

### Arguments

| Flag | Required | Description |
|------|----------|-------------|
| `--commit SHA` | ✅ | Expected commit (verification) |
| `--run-id ID` | ✅ | GitHub run id for logging |

### Environment (from server shell)

| Variable | Source |
|----------|--------|
| `STAGING_APP_PATH` | Passed from SSH or default `$PWD` |

### Steps (MUST execute in order)

1. **Preflight**
   - `set -euo pipefail`
   - Verify `node`, `npm`, `pm2`, `git` exist
   - `cd "$STAGING_APP_PATH"`

2. **Update code**
   ```bash
   git fetch origin main
   git reset --hard "$COMMIT_SHA"   # or origin/main if matches
   ```

3. **Install dependencies**
   ```bash
   npm ci
   ```

4. **Load environment**
   ```bash
   test -f .env || { echo "Missing .env"; exit 1; }
   set -a && source .env && set +a
   ```

5. **Database schema**
   ```bash
   npm run db:push
   ```
   On failure → exit 1 (no pm2 reload).

6. **Build**
   ```bash
   npm run build
   ```
   On failure → exit 1.

7. **Reload application**
   ```bash
   pm2 reload gpi-staging --update-env
   ```
   Or if first run: `pm2 start ecosystem.config.cjs`

8. **Log deploy**
   ```bash
   echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) run=$RUN_ID sha=$COMMIT_SHA status=success" >> logs/deploy.log
   ```

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Any step failed |
| 2 | Preflight / missing .env |

### Idempotency

Re-running script on same commit: safe (git reset, npm ci, rebuild, reload).

## nginx (reference config snippet)

Not deployed by CI; documented for ops:

```nginx
server {
    listen 443 ssl http2;
    server_name staging.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## Media persistence

`public/media` or Payload upload dir MUST survive deploy:

- Directory gitignored
- Deploy script MUST NOT `git clean -fdx` on media paths

Recommended:

```bash
git clean -fdx -e public/media -e .env -e logs/
```

Only if clean step added — default script avoids destructive clean.

## Rollback (manual, v1)

1. GitHub → Actions → `deploy-staging` → Run workflow
2. Input ref = previous commit SHA
3. Or SSH: `git reset --hard <sha> && bash scripts/deploy/staging-remote.sh ...`
